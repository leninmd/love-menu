# 恋上菜单 - 实施计划

> 基于《恋上菜单（Love Menu）项目综合文档》V1.0  
> 创建日期：2026-04-30  
> 状态：待执行

## 阶段概览

- 阶段 1：项目基础结构搭建
- 阶段 2：数据库与认证系统
- 阶段 3：店主核心功能
- 阶段 4：顾客核心功能
- 阶段 5：通用功能与部署

## 详细任务分解

### 阶段 1：项目基础结构搭建
1. 产出目标目录结构清单（client/server/data 及子目录）  
   - 优先级：高  
   - 依赖：无  
2. 明确前端路由页面列表（管理端/顾客端/登录/订单等）  
   - 优先级：高  
   - 依赖：1  
3. 明确后端模块清单（认证/餐厅/菜品/订单/推送/评价）  
   - 优先级：高  
   - 依赖：1  
4. 定义全局配置项清单（环境变量与用途）  
   - 优先级：中  
   - 依赖：1  
5. 明确图片存储与访问路径规则（本地磁盘/静态服务）  
   - 优先级：中  
   - 依赖：3  
6. 输出阶段验收清单（“结构与边界已确认”）  
   - 优先级：中  
   - 依赖：2,3,4,5  

### 阶段 2：数据库与认证系统
1. 列出数据模型与字段清单（users、credentials、restaurants 等）  
   - 优先级：高  
   - 依赖：阶段1-3  
2. 明确软删除与订单快照规则（含边界场景）  
   - 优先级：高  
   - 依赖：1  
3. 输出认证流程说明（通行密钥/邮箱兜底/游客访问）  
   - 优先级：高  
   - 依赖：阶段1-3  
4. 明确登录态与权限矩阵（店主/顾客/游客）  
   - 优先级：高  
   - 依赖：3  
5. 定义验证码限流与错误锁定规则  
   - 优先级：中  
   - 依赖：3  
6. 输出阶段验收清单（“模型与认证规则已冻结”）  
   - 优先级：中  
   - 依赖：1,2,3,4,5  

### 阶段 3：店主核心功能
1. 列出餐厅管理需求清单（创建/编辑/软删除/分享）  
   - 优先级：高  
   - 依赖：阶段2-4  
2. 列出分类管理规则（排序/不可删除条件）  
   - 优先级：高  
   - 依赖：1  
3. 列出菜品维护规则（字段、图片压缩、来源平台）  
   - 优先级：高  
   - 依赖：1  
4. 明确订单处理流程（接单/完成/通知）  
   - 优先级：高  
   - 依赖：阶段2-4  
5. 定义食客等级规则（多级、最少一条、排序）  
   - 优先级：中  
   - 依赖：1  
6. 输出阶段验收清单（“店主功能边界确认”）  
   - 优先级：中  
   - 依赖：1,2,3,4,5  

### 阶段 4：顾客核心功能
1. 列出菜单浏览与搜索规则（分类/实时搜索）  
   - 优先级：高  
   - 依赖：阶段3-3  
2. 明确来源图标交互规则（复制菜名/Toast）  
   - 优先级：高  
   - 依赖：1  
3. 定义购物车行为（数量修改/跨页保留/断网提示）  
   - 优先级：高  
   - 依赖：1  
4. 明确下单快照逻辑与删除菜品校验  
   - 优先级：高  
   - 依赖：阶段2-2,3  
5. 明确订单跟踪与状态展示规则  
   - 优先级：中  
   - 依赖：4  
6. 明确评价约束（仅已完成订单、字数限制）  
   - 优先级：中  
   - 依赖：5  
7. 输出阶段验收清单（“顾客功能边界确认”）  
   - 优先级：中  
   - 依赖：1,2,3,4,5,6  

### 阶段 5：通用功能与部署
1. 明确推送订阅与发送规则（Web Push/VAPID）  
   - 优先级：高  
   - 依赖：阶段3-4,4-5  
2. 明确消息中心聚合规则（店主/顾客视图）  
   - 优先级：中  
   - 依赖：阶段3-4,4-5  
3. 明确 PWA 与离线策略（缓存范围/离线提示）  
   - 优先级：中  
   - 依赖：阶段1-2  
4. 明确部署与持久化规则（Railway/Volume/环境变量）  
   - 优先级：高  
   - 依赖：阶段1-4  
5. 明确测试分发方式与限制（ngrok/分享链接）  
   - 优先级：低  
   - 依赖：4  
6. 输出阶段验收清单（“通用能力与部署规则确认”）  
   - 优先级：中  
   - 依赖：1,2,3,4,5
