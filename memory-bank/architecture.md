```
└─ data/                  # SQLite 持久化数据（Railway Volume）
```

## 模块职责与边界（规划）

- client/: 菜单浏览、下单、店主管理、PWA/离线缓存、推送订阅与接收。
- server/: 认证（通行密钥/邮箱）、餐厅与菜品管理、订单流转、评价、推送发送、图片静态服务。
- data/: SQLite 数据文件与持久化卷挂载路径。

## 关键依赖与运行形态（规划）

- 前端：Vue 3 + Vite + Pinia + Vue Router + TailwindCSS + vite-plugin-pwa。
- 后端：Node.js + Express + Drizzle ORM + SQLite。
- 第三方：Brevo/Resend（邮件）、Web Push（VAPID）。

## 数据与快照策略（规划）

- 订单菜品快照独立存储（不依赖菜品表外键），保障删除后可追溯。
- 餐厅/菜品软删除，订单与历史记录保留。

## 阶段 2 细化产出（已确认）

### 数据模型清单（字段/约束/关系）

- users：`id`、`email (unique)`、`nickname`、`avatar_url`、`created_at`
- credentials：`id`、`user_id (FK)`、`public_key`、`counter`、`device_name`、`created_at`
- restaurants：`id`、`owner_id (FK)`、`name`、`avatar_url`、`intro`、`is_deleted`、`created_at`
- categories：`id`、`restaurant_id (FK)`、`name`、`sort_order`、`created_at`
- dishes：`id`、`restaurant_id (FK)`、`category_id (FK nullable)`、`name`、`image_url`、`description`、`price`、`sources (JSON)`、`is_deleted`、`order_count`、`created_at`
- guest_levels：`id`、`restaurant_id (FK)`、`title`、`min_orders`、`sort_order`
- carts：`id`、`user_id (FK)`、`restaurant_id (FK)`、`created_at`
- cart_items：`id`、`cart_id (FK)`、`dish_id (FK)`、`quantity`
- orders：`id`、`customer_id (FK)`、`restaurant_id (FK)`、`status`、`total_price`、`snapshot_at`、`created_at`
- order_items：`id`、`order_id (FK)`、`dish_name`、`dish_price`、`dish_sources`、`dish_image`、`quantity`、`is_reviewed`、`original_dish_id`
- reviews：`id`、`order_item_id (FK unique)`、`user_id (FK)`、`content`、`created_at`

### 软删除与快照规则矩阵

- 餐厅删除：软删除，外部访问显示“已下线”，历史订单保留快照。
- 菜品删除：软删除，已下单快照继续显示。
- 账户注销：
  - 店主：餐厅与菜品标记下线/软删除，订单快照保留。
  - 顾客：历史订单快照保留，但账户数据删除。
- 订单快照：`order_items` 独立存储菜名、价格、来源、图片，不依赖 `dishes` 外键。

### 认证流程说明

- 主路径：通行密钥注册/登录 → 获取 JWT → 正常访问。
- 兜底路径：邮箱验证码登录 → 获取 JWT → 正常访问。
- 游客访问：无 JWT，仅允许菜单浏览/搜索类接口。
- 异常回退：
  - WebAuthn 不支持 → 隐藏通行密钥入口 → 强制邮箱登录。
  - 通行密钥注册失败 → 提示失败并回退邮箱登录表单。

### 权限矩阵

- 游客：浏览餐厅菜单、搜索菜品（只读）。
- 顾客：购物车/下单/订单跟踪/菜品评价/消息中心（顾客视图）。
- 店主：餐厅/菜品/分类/订单处理/食客等级/消息中心（店主视图）。
- 所有写操作：需 JWT 登录。
- 订单状态变更：仅店主。

### 验证码与限流规则清单

- 发送频率：同邮箱 1 分钟 1 次。
- 每日上限：同邮箱每日发送上限。
- 错误锁定：连续 5 次错误后当日禁止获取验证码。
- 解锁：次日自动解锁。

## 阶段 3 细化产出（已确认）

### 餐厅管理规则（店主）

- 创建：名称必填；头像/简介可选。
- 编辑：支持更新名称/头像/简介。
- 删除：软删除；分享链接访问显示“该餐厅已下线”。
- 分享：生成专属链接与二维码（复制/保存）。

### 分类管理规则

- 可新增/编辑/排序。
- 分类下有菜品时不可删除。
- 空分类允许存在（便于提前规划菜单结构）。

### 菜品维护规则

- 必填字段：菜名。
- 图片：拍照/相册，前端压缩至 ≤300KB。
- 描述/展示价格可选。
- 来源平台：美团/美团外卖/京东到家/京东/抖音/淘宝，多选 + 自定义来源。
- 删除：软删除，已下单快照继续显示。
- 统计：菜品被点次数自动累加。

### 订单处理流程

- 状态流转：已提交 → 已接单 → 已完成。
- 操作：店主可“接单”“标记完成”。
- 通知：状态变化触发推送。
- 异常：无超时自动取消；长期停滞保持原状态。

### 食客等级规则

- 店主可设置多级称号。
- 每级配置最低下单次数。
- 至少保留一条记录。
- 排序与展示由店主定义。

### 菜单浏览与搜索规则（顾客）

- 通过分享链接/二维码进入餐厅。
- 按分类浏览菜品卡片。
- 支持按菜品名称实时搜索。

### 来源图标交互规则

- 点击来源图标复制菜品名称到剪贴板。
- Toast 提示复制成功。
- 不进行任何外部跳转。

### 购物车与下单规则

- 购物车支持数量修改与删除。
- 断网时保留购物车内容并提示。
- 下单生成不可变订单快照（菜名/价格/来源）。
- 下单时若菜品已被删除，后端返回错误并移除无效项。
- 防重复提交与频率限制。

### 订单跟踪规则

- 状态展示：已提交 → 已接单 → 已完成。
- 状态变更触发推送通知。

### 菜品评价规则

- 仅已完成订单的菜品可评价。
- 文字评价最多 200 字。
- 评价展示在菜品详情页。

### 消息中心规则（顾客视图）

- 展示“我提交的订单”。
- 支持按类型筛选。
